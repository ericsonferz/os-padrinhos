# Recurrence

## Objetivo

Representa padrões de repetição.

Hoje possui pouca utilização, mas prepara o domínio para futuras funcionalidades.

---

## Exemplos

- reunião semanal
- pagamento mensal
- lembrete diário

---

## Utilizado por

- Tasks
- Notifications
- Payments (futuro)

---

## Atributos

- frequency
- interval
- endCondition

---

## Responsabilidades

Definir quando uma ação se repete.

---

## Invariantes

É opcional.

Nunca altera a entidade que o utiliza.

É imutável.
