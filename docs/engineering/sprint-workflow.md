# Sprint Workflow

## Objetivo

Definir oficialmente o ciclo de desenvolvimento utilizado no projeto.

Cada Sprint representa uma unidade completa de implementação.

Nunca avançamos para a Sprint seguinte sem concluir integralmente a atual.

---

# Fluxo

Planeamento

↓

Documentação

↓

RFC

↓

Implementação

↓

Testes

↓

Progress Report

↓

Auditoria

↓

Merge

---

# Implementação

Cada Sprint deverá produzir:

- código
- testes
- documentação
- atualização do AI Knowledge Base
- atualização do README (quando necessário)

---

# Critérios de Conclusão

Uma Sprint apenas é considerada concluída quando:

- compila
- todos os testes passam
- cobertura mínima atingida
- documentação atualizada
- auditoria concluída
- aprovação final

---

# Sprint Packages

O desenvolvimento será organizado por Sprint Packages.

Cada Package deverá conter entre cinco e dez ficheiros relacionados.

O objetivo é manter o projeto sempre num estado funcional.

---

# Relatórios

No final de cada fase deverá ser produzido um:

Architecture & Implementation Progress Report

O relatório será armazenado em:

docs/reports/

---

# Convenção

Nome dos relatórios:

AAAA-MM-DD-HHMM-report.md

Exemplo:

2026-07-22-1500-report.md
